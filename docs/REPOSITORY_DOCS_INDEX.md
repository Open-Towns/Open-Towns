# How to add repo docs

When adding a new documentation file:

1. Choose the correct documentation category:
   - `docs/getting-started/` for setup, build, run, and first-time contributor docs.
   - `docs/modding/` for modding, Towns++, data override, and mod compatibility docs.
   - `docs/systems/` for code systems, managers, architecture, gameplay logic, and implementation details.
2. Add the file under the matching category folder.
3. Use a clear filename that matches the local style of that folder.
4. Start the file with a heading and a short first paragraph that explains the document purpose.
5. Let this workflow update the Repository Docs Index automatically.
6. Use the Wiki for navigation, summaries, and contributor-friendly explanations.

If a document does not clearly fit one of the current categories, discuss whether a new category is needed before adding it.

# Documentation rule

Use this layout:

```text
README = short project overview
docs/ = technical code documentation
Wiki = documentation index, summaries, and contributor navigation
```
